Code Analysis for software-based artworks

Software-based art works, such as Still Life, datamatics, or Subtitled Public are made of multiple layers of code to produce the artwork, control light, randomize some parts, fetch data, synthesize sound, control interactions, calibrate projectors, etc. These different layers are made with various programming languages (e.g., Java applets for web-based pieces in the 2000's, javascript, rust, and python for more recent pieces), are optimized for certains resolutions or compiled for certain runtime environments. Consequently, these pieces of art can break over time, because the libraries on which it depends evolve, because runtime platforms change, or because certain technologies disappear (e.g. Java applets).

In this work, we investigate code analysis techniques for software-based artworks to build a consolidated body of knowledge about the core libraries, compilers and frameworks used in these artworks; the ability to automatically port them on different platforms; and to design novel techniques to migrate art code to recent architectures or package it into isolated, reproducible containers (e.g., with Webassembly).

Examples of relevant, state of the art computer science concepts to support digital art include:

  • Code transformation. Many artworks rely on third-party libraries (e.g. p5.js>) or APIs (e.g. canvas), which evolve fast. In this project we aim at designing sound source-to-source transformations that support the co-evolution of artworks and their runtime environment.
  • Containers. A key challenge of digital artworks is to thoroughly document all its runtime dependencies. Recent work explore container-based solution to bundle artworks and their dependencies, make them more reproducible. In this project we study Webassembly to revive old artworks, e.g. pieces based on Java applets, and to bundle contemporary browser-based artworks.
  • Unikernels. When they rely on large sets of dependencies and drivers, a complete setup for an artwork can grow very large. Yet, once the artwork is delivered in a gallery, it can live in a close world. This is an opportunity to tailor the artwork's environment to its exact needs and remove large portions of code bloat. In this project, we explore the opportunity of building unikernels for low-resources artworks.
  • Energy harvesting. Several artists explore low-fi technology to build robust and sustainable artworks, which can be exhibited outside gallery spaces, e.g. in public outdoor spaces or in natural spaces. In this project, we explore energy harvesting techniques, for the construction of low-power digital artworks.

The research work will be performed at the Université de Montréal, under the supervision of Benoit Baudry, and in collaboration with artists and museums in Montreal. It can be a Master's or PhD thesis position.
